===DOCUMENT: transformer_architecture===
The Transformer architecture was introduced by Vaswani et al. in the 2017 paper "Attention Is All You Need". The model relies entirely on self-attention mechanisms to compute representations of its input and output, replacing traditional recurrent neural networks. Transformers consist of an encoder stack and a decoder stack, each made of multi-head attention layers and feed-forward networks. The key innovation is the multi-head attention mechanism, which allows the model to jointly attend to information from different representation subspaces. Positional encodings are added to input embeddings because the model contains no recurrence or convolution. The Transformer was originally designed for machine translation but became the foundation for large language models like GPT and BERT.

===DOCUMENT: bert_model===
BERT, which stands for Bidirectional Encoder Representations from Transformers, was developed by Google AI in 2018. BERT is pre-trained on two tasks: masked language modeling and next sentence prediction. Unlike GPT, BERT uses a bidirectional approach, meaning it reads text in both directions simultaneously. BERT was trained on the entire Wikipedia corpus and the BookCorpus dataset. Fine-tuning BERT on downstream tasks like question answering, named entity recognition, and text classification achieves state-of-the-art results. The model uses WordPiece tokenization and supports a maximum sequence length of 512 tokens. BERT influenced many subsequent models including RoBERTa, DistilBERT, and ALBERT.

===DOCUMENT: gpt_series===
The GPT (Generative Pre-trained Transformer) series was developed by OpenAI. GPT-1 was released in 2018 and demonstrated that language models pre-trained on large corpora could be fine-tuned on downstream tasks with minimal labeled data. GPT-2 followed in 2019, scaling to 1.5 billion parameters and showing emergent abilities in text generation. GPT-3, released in 2020, contained 175 billion parameters and demonstrated few-shot learning capabilities without any fine-tuning. GPT-4, released in 2023, is a multimodal model capable of processing both text and images. OpenAI trained GPT models using unsupervised pre-training on diverse internet text followed by reinforcement learning from human feedback (RLHF). The GPT architecture uses a decoder-only Transformer with causal self-attention masking.

===DOCUMENT: rag_systems===
Retrieval-Augmented Generation (RAG) was introduced by Lewis et al. at Facebook AI Research in 2020. RAG combines parametric memory from a pre-trained language model with non-parametric memory via a dense retrieval component. The retrieval component uses a Dense Passage Retriever (DPR) to fetch relevant documents from an external knowledge base. The generator then conditions on both the input query and the retrieved documents to produce an answer. RAG outperforms pure parametric models on knowledge-intensive NLP tasks like open-domain question answering. Advanced RAG techniques include query rewriting, re-ranking, and hybrid sparse-dense retrieval. Graph RAG extends the approach by structuring the knowledge base as a graph, enabling multi-hop reasoning across related entities.

===DOCUMENT: knowledge_graphs===
A knowledge graph is a structured representation of information where entities are nodes and relationships are edges. Knowledge graphs are used by major companies including Google, Microsoft, and Amazon to power search and recommendation systems. The Google Knowledge Graph, introduced in 2012, contains billions of facts about real-world entities. Neo4j is the most popular graph database for storing and querying knowledge graphs. In the context of NLP, knowledge graphs can be constructed automatically using information extraction techniques. Relation extraction models identify subject-predicate-object triples from unstructured text. Knowledge Graph Embeddings like TransE and RotatE learn vector representations of entities and relations to support reasoning. NetworkX is a Python library widely used for constructing and analyzing graphs in research settings.

===DOCUMENT: vector_databases===
Vector databases are specialized storage systems designed to handle high-dimensional vector embeddings efficiently. Embeddings are dense numerical representations of data produced by neural networks. Popular vector databases include Chroma, Pinecone, Weaviate, Qdrant, and FAISS. Chroma is an open-source embedding database that runs locally and integrates with LangChain. Pinecone is a managed cloud vector database that supports real-time updates and hybrid search. FAISS, developed by Facebook AI Research, is a library for efficient similarity search over dense vectors. Vector databases enable semantic search by finding nearest neighbors in embedding space using metrics like cosine similarity and Euclidean distance. They are a core component of RAG pipelines.

===DOCUMENT: langchain_framework===
LangChain is an open-source framework for building applications powered by large language models. It was created by Harrison Chase and first released in 2022. LangChain provides abstractions for chains, which are sequences of calls to LLMs and other tools. The framework supports retrieval-augmented generation through its RetrievalQA and ConversationalRetrievalChain components. LangChain integrates with numerous vector stores, document loaders, and embedding providers. LCEL (LangChain Expression Language) allows composing chains using a pipe operator syntax. LangChain also provides LangSmith, a platform for tracing, debugging, and evaluating LLM applications. The framework works with models from OpenAI, Anthropic, Hugging Face, Ollama, and many other providers.

===DOCUMENT: ollama_local_llms===
Ollama is an open-source tool that allows users to run large language models locally on their own hardware. It was released in 2023 and supports models including Llama 3, Mistral, Gemma, Phi, Qwen, and DeepSeek. Ollama provides a simple REST API compatible with the OpenAI API specification. Users can pull models using a single command and run inference without any cloud dependency. Ollama supports GPU acceleration via CUDA and Metal, enabling fast inference on NVIDIA and Apple Silicon hardware. The tool manages model storage, quantization, and context window configuration automatically. Ollama integrates natively with LangChain through the OllamaLLM and OllamaEmbeddings classes.

===DOCUMENT: ragas_evaluation===
RAGAS (Retrieval Augmented Generation Assessment) is a framework for evaluating RAG pipelines without requiring human-annotated ground truth for every metric. It was developed by Shahul Es et al. and released as an open-source Python library. RAGAS measures several key metrics: faithfulness measures whether the generated answer is grounded in the retrieved context; answer relevancy measures how relevant the answer is to the original question; context recall measures how much of the ground truth is covered by the retrieved context; context precision measures the signal-to-noise ratio of the retrieved documents. RAGAS uses LLMs internally to judge these metrics. The framework integrates with LangChain and LlamaIndex and supports batch evaluation over datasets. Iterating on retrieval strategy based on RAGAS scores is a standard practice for improving RAG pipeline quality.

===DOCUMENT: graph_neural_networks===
Graph Neural Networks (GNNs) are a class of neural network architectures designed to operate on graph-structured data. GNNs were formalized by Scarselli et al. in 2009 and gained prominence with the development of Graph Convolutional Networks (GCN) by Kipf and Welling in 2017. Message passing is the core computational paradigm in GNNs, where nodes aggregate information from their neighbors iteratively. GraphSAGE introduces inductive learning by sampling fixed-size neighborhoods. Graph Attention Networks (GAT) use attention mechanisms to weight neighbor contributions. GNNs are applied to molecular property prediction, social network analysis, recommendation systems, and knowledge graph completion. PyTorch Geometric and DGL are the leading libraries for implementing GNNs.

===DOCUMENT: embeddings_and_semantic_search===
Word embeddings are dense vector representations of words that capture semantic meaning. Word2Vec, developed by Mikolov et al. at Google in 2013, produces word embeddings using shallow neural networks trained with skip-gram or CBOW objectives. GloVe, developed at Stanford, produces embeddings by factoring global word co-occurrence statistics. Sentence embeddings extend the idea to represent full sentences as fixed-size vectors. Sentence-BERT (SBERT) produces semantically meaningful sentence embeddings using siamese BERT networks fine-tuned on natural language inference tasks. The all-MiniLM-L6-v2 model from sentence-transformers is a popular lightweight embedding model. Nomic Embed and mxbai-embed-large are high-quality embedding models available through Ollama. Cosine similarity between embedding vectors is the standard metric for semantic search and retrieval.
